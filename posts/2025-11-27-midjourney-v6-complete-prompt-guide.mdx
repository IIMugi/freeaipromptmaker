# Midjourney V6.1 Parameter Reference (Legacy Model)

**V8.1 is the current default; V6.1 is a legacy model.** Use this page only when a record must intentionally select V6.1. For a new unversioned prompt, consult current Midjourney documentation instead of assuming V6.1 behavior.

V6.1 was released on 2024-07-30 and was the default until 2025-06-16. **Documentation snapshot checked: 2026-07-16.** The official Version article was modified 2026-06-16, and the official Parameter List was modified 2026-06-11 when checked.

**Documented:** V8.1 is the current default, V6.1's release/default dates, the `--v` version selector, required parameter placement, and the current Version page's V6-family compatibility chart.

**Locally checked:** the text-only parser example and migration worksheet were checked for spacing, ordering, and exact version text on 2026-07-16.

**Not tested:** no prompt was submitted and no output, parameter behavior, or cross-version image result was compared. No independent image-generation comparison was performed for this guide.

## Select V6.1 explicitly

The official Version page says to add the version parameter at the end of the prompt. For this legacy model, use:

```text
--v 6.1
```

V8.1 is the platform default, but the official Version page also allows a user to choose an account-level default in settings. Unversioned prompts are state-dependent and nonportable. Pin `--v 6.1` whenever the record must reproduce the legacy model choice.

## Prompt text versus parameters

The current Parameter List defines a strict text format:

- describe the requested content first;
- put parameters at the end;
- leave a space between prompt text and the two hyphens;
- do not put punctuation inside a parameter;
- do not resume prompt prose after parameters begin.

### Reproducible text-only placement example

```text
quiet reading room, oak shelves, afternoon window light --v 6.1
```

This can be checked without generating anything:

| Segment | Parsed role | Check |
|---|---|---|
| `quiet reading room, oak shelves, afternoon window light` | Prompt text | Appears before all parameters |
| one space | Separator | Exists before the first `--` |
| `--v 6.1` | Version parameter | Appears at the end and selects V6.1 |

Incorrect placement examples:

```text
quiet reading room--v 6.1
quiet reading room --v 6.1, afternoon light
quiet reading room - - v 6.1
```

The first lacks the documented separator, the second returns to prompt text after a punctuated parameter, and the third splits the two hyphens.

## V6.1 compatibility table

The supplied Version page labels its comparison column **V6**, not **V6.1**. This guide does not silently convert that family-level label into V6.1-specific confirmation. The table therefore distinguishes the one exact V6.1 selection shown by the documentation boundary from parameters that the current pages list only generally or under the V6-family column.

| Parameter or feature | Exact V6.1 confirmation in the supplied pages | Treatment in this reference |
|---|---|---|
| `--v 6.1` | Yes: the Version page documents `--v #` and identifies V6.1 as an available historical version | Use to select the legacy model |
| `--ar`, `--chaos`, `--no`, `--seed`, `--raw`, `--stylize`, `--tile`, `--weird` | No exact V6.1 label; the compatibility chart marks these in its **V6** column | Not assigned a V6.1 range or behavior here |
| `--q` | No exact V6.1 label; the chart shows values `0.5, 1, 2` for its **V6** column | Do not present that family range as V6.1-specific confirmation |
| `--hd` | No; the current Parameter List describes it for V8.1, and the Version chart does not mark HD images for V6 | Remove from a V6.1 record |
| `--draft` | No; the current Parameter List describes Draft for V7 | Remove from a V6.1 record |
| `--oref` | No; the Version chart does not mark Omni Reference for V6 | Remove from a V6.1 record |
| `--exp` | No; the Version chart does not mark it for V6 | Remove from a V6.1 record |

This narrow table is intentional. Parameter names on a current all-version list do not establish their accepted values or semantics in V6.1. Use a version-specific primary page before adding any additional range to a V6.1 production worksheet.

## Migration checklist

Use this when converting current prompt text into a deliberately legacy V6.1 record:

1. Copy only the descriptive prompt text.
2. Remove current-version carryover that is not confirmed for V6.1, including `--hd`, `--draft`, `--oref`, and `--exp`.
3. Remove any remembered range or default that came from a V8.1 page.
4. Put `--v 6.1` at the end.
5. Confirm there is a space before the two hyphens and no prompt prose after the parameter.
6. If another parameter is required, find an official page that explicitly identifies V6.1 compatibility; otherwise leave it out of the reproducible record.
7. Save the documentation snapshot date with the prompt.

## One-variable troubleshooting

1. Start with prompt text plus only `--v 6.1`.
2. Check placement and spacing as text.
3. If Midjourney rejects the prompt, remove any unverified carryover parameter before changing the prompt text.
4. Add back at most one parameter whose V6.1 support is explicitly documented by a current official source.
5. Record that source and its check date beside the parameter.

This procedure diagnoses syntax and provenance. It does not establish output behavior.
